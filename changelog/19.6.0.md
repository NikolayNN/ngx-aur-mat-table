# 19.6.0

Feedback-batch release: six additive features, one breaking API change and one
visual default change. Suite green (137 tests), library and demo build clean.

## Breaking: `highlightClicked` removed — use `clickCfg.styleCfg`

`ClickConfig.highlightClicked` is gone. The click-highlight style/class now
lives in `clickCfg.styleCfg`, mirroring the existing `hoverCfg.styleCfg` shape
(`class` + `style`):

```ts
// before
clickCfg: { highlightClicked: StyleBuilder.Row.builder().background('blue'), cancelable: true }

// after
clickCfg: { styleCfg: { style: StyleBuilder.Row.builder().background('blue') }, cancelable: true }
```

New in the same shape: `styleCfg.class` puts a CSS class on the highlighted
`<tr>` — class-based highlighting without `StyleBuilder`:

```ts
clickCfg: { styleCfg: { class: 'row-selected' }, cancelable: true }
```

```css
tr.row-selected { background: #e3f2fd; }
tr.row-selected td { color: #0d47a1; } /* cell text color is the consumer's selector */
```

The internal `new-color` helper (cell text color inheritance) stays tied to the
`style` branch only; a class-based highlight styles cells through its own
selectors, as above.

## Added: column alignment — `align`

`ColumnConfig.align` and `IndexConfig.align` (`'left' | 'center' | 'right'`,
exported type `ColumnAlign`) align the column header, its body cells and its
total-row cell in one go. Default `'left'` keeps today's rendering. Works for
all three cell kinds: plain text cells, `lib-column-view` content (flex) and
sortable headers (Material's `.mat-sort-header-container`).

```ts
columnsCfg: [{ name: 'Price', key: 'price', valueConverter: v => v.price, align: 'right' }],
indexCfg: { align: 'center' },
```

## Added: index formatter

`IndexConfig.formatter` formats the rendered row index (offset already
applied) — `1.` instead of `1`, `№1`, zero-padding, anything:

```ts
indexCfg: { offset: 1, formatter: i => `${i}.` }
```

## Added: `ColumnSize.fit` — size column to content

`size: { fit: true }` shrinks a column to its content (`width: 1%` +
`white-space: nowrap` — the reliable table idiom, no more `width: '1%'` hacks
in consumer code). Available on data columns and every special column that
takes a `ColumnSize` (index, selection, actions, drag, timeline). Don't combine
with `width` — an explicit inline width wins.

## Added: keyboard accessibility for clickable rows

When `bodyRowCfg.clickCfg` is configured, rows are now keyboard-accessible
automatically — no new flags:

- `tabindex="0"` — rows reachable with Tab (WCAG 2.1.1);
- Enter/Space activate the row exactly like a click (`rowClick`, highlight,
  `cancelable` toggling); Space doesn't scroll the page; Enter/Space on nested
  controls (checkboxes, action buttons) do **not** double-activate the row;
- `aria-current="true"` marks the highlighted row for screen readers (also with
  the external `[highlight]` input), so selection is no longer color-only
  (WCAG 1.4.1, 4.1.2);
- a visible `:focus-visible` outline (WCAG 2.4.7).

Read-only tables (no `clickCfg`) are untouched — no tab stops added.

## Added: configurable cell padding — CSS variables + config

Horizontal cell padding is now driven by CSS custom properties with the old
values as defaults: `--aur-cell-padding-left` / `--aur-cell-padding-right`
(default `4px`). Set them from config or plain CSS at any level — no more
fighting the library's `!important` (it remains only to beat Material MDC's
own cell padding; the *value* is yours):

```ts
tableViewCfg: { cellPaddingLeft: '8px', cellPaddingRight: '8px' },   // whole table
columnsCfg: [{ ..., size: { paddingRight: '12px' } }],               // single column (wins over table)
```

```css
/* or pure CSS on any ancestor */
.my-dense-table { --aur-cell-padding-left: 2px; --aur-cell-padding-right: 2px; }
```

### Behavior change: last-header 25px only with the settings button

The unconditional `padding-right: 25px` on the last header cell existed to keep
header text clear of the absolutely-positioned settings button
(`headerButtonCfg`). It is now applied **only when the header button is
enabled** (and is tunable via `--aur-last-header-padding-right`). Tables
without the button gain those 25px back in the last column — check layouts
that relied on the extra gap.

## Added: icon `tooltipClass` and `position`

`IconView` gets two fields:

- `tooltipClass` — CSS class(es) forwarded to `matTooltipClass`, on cell/header
  icons and on action-button tooltips. Resolver-typed like the other icon
  fields, so per-row classes work: `tooltipClass: r => `tt-${r.rowSrc.severity}``.
  The tooltip renders in the global overlay — define the class in global styles.
- `position: 'start' | 'end'` — icon before (default) or after the cell text.
  Note for long-time users: an unused `IconView.position` (`'right' | 'left'`)
  was removed in 19.2.0; this is a new, actually-rendered option with the
  unified `start/end` vocabulary. Applies in cells/headers; action buttons and
  the drag icon ignore it.

```ts
valueView: {
  icon: {
    name: () => 'warning',
    tooltip: r => r.rowSrc.hint,
    tooltipClass: () => 'my-tooltip',
    position: 'end',
  },
}
```
