# 19.12.0

Single additive feature, no breaking changes: a table can now have **several**
action columns instead of one. `actionCfg` accepts either a single config (as
before) or an array, each entry an independent column with its own `key`,
`position`, `size` and `actions`. A single object still produces the legacy
`tbl_actions` column unchanged. Suite green (264 tests), library and demo build
clean.

## Added: multiple action columns — `actionCfg` array + anchor positioning

`actionCfg` widens to `ActionConfig<T> | ActionConfig<T>[]`. Each array entry is
one action column with a unique `key` (which doubles as the column's
`matColumnDef` name) and its own `position`, `size` and `actions`:

```ts
import { AUR_COLUMN, TableConfig } from 'ngx-aur-mat-table';

actionCfg: [
  { key: 'primary', position: 'start',                        actions: [ /* edit */ ] },
  { key: 'tools',   position: { after: 'email' },             actions: [ /* copy */ ] },
  { key: 'manage',  position: { before: AUR_COLUMN.selection }, actions: [ /* archive (menu) */ ] },
]
```

`position` extends from `'start' | 'end'` to also accept anchors
`{ before: key }` / `{ after: key }`, placing a column immediately before or after
any other column — a data column (by its `key`), another action column, or a
special column via the new **`AUR_COLUMN`** constant (`selection` / `index` /
`drag` / `timeline`). Order resolves in two phases: `start`/`end` columns first
(unchanged from before), then anchors, iteratively so chains
(`{ after: anotherActionKey }`) resolve once their target lands. An anchor that
points at a missing column degrades to `end` with a dev-mode warning; a duplicate
or colliding `key` drops that column with a dev-mode warning. For full manual
control `[displayColumns]` still wins — list each action column by its `key`.

Back-compat is total: a single `actionCfg: { … }` object keeps working untouched
and still produces the `tbl_actions` column, so existing `[displayColumns]`
referencing `'tbl_actions'` is unaffected. `key` is optional and defaults to
`tbl_actions`; `position` defaults to `'end'`. New public exports: `AUR_COLUMN`,
`ActionColumnPosition`, `ActionColumnView`.
