# 19.7.0

Feedback-batch release: nine additive features and fixes, no breaking changes.
Server-mode tables get the bigger share (sorting, initial sort state, a
page-loaded event, first/last buttons, correct row numbering); rows, actions
and icons get per-row styling, disabled actions and tooltip positioning. Suite
green (185 tests), library and demo build clean.

## Fixed: server pages are no longer re-sorted locally

In server wiring (`pageSource` or `paginationCfg.mode: 'server'`) the table no
longer binds `MatSort` to its `MatTableDataSource`, so a loaded server page is
rendered exactly in the order the server returned it. Previously a header click
re-sorted the current page in-memory by `valueConverter` values — a visible
"jump" before the response arrived, and the server order was overwritten on
arrival (dates formatted as `dd.MM.yyyy` sorted lexicographically, etc.). The
sort arrows and the `(sort)` event are unaffected (they live on the `MatSort`
directive). `ColumnConfig.sort.customSort` is ignored in server mode.

> Behavior change for `pageSource` tables with an active sort: rows now follow
> the server's order rather than the local re-sort.

## Added: initial sort state — `tableConfig.sortCfg`

`sortCfg: { active, direction }` lights the header sort arrow from the start and
makes the first `pageSource` request carry that sort (`req.sort`), with no
duplicate request (initialization does not emit `matSortChange`). Client-mode
tables sort their data on init too.

```ts
sortCfg: { active: 'id', direction: 'desc' }
```

This is initial state, not a reactive control: changing it at runtime moves the
arrow but does not re-sort or refetch. Without `sortCfg` behavior is unchanged.

## Added: `(pageLoaded)` event

A new `@Output() pageLoaded` emits the applied server page so the host can drive
header counters, charts and related views without wrapping `pageSource` in a
`tap()`:

```ts
// AurPageLoadedEvent<T>
(pageLoaded)="onPageLoaded($event)"   // { content, totalElements, pageIndex }
```

Emitted after the table state is applied, on every successful load (start, page
change, sort, `reload()`). Errors still go to `pageError`; there is no event in
manual/legacy/client modes. Completes the load-lifecycle trio with
`loadingChange` and `pageError`.

## Added: `paginationCfg.showFirstLastButtons`

Controls the built-in paginator's jump-to-first/last buttons (default `true`,
i.e. shown as before). Bound straight to the config rather than the internal
provider snapshot, so swapping the `tableConfig` reference at runtime (e.g. a
breakpoint observer hiding them on small screens) toggles the buttons without a
data rebuild. An `externalPaginator` is owned by the host and unaffected.

```ts
paginationCfg: { enable: true, size: 20, mode: 'server', showFirstLastButtons: false }
```

## Fixed: absolute row index across pages in server mode

With `indexCfg` in server (`pageSource`) mode the index column showed `1..N` on
every page, because `tableData` holds only the current page and row ids are
0-based within it. The index now adds the page offset (`pageIndex * pageSize`),
so page 2 continues `21, 22, …`. Client-mode pagination is unaffected — its ids
already span the full dataset. The offset is gated on `paginatorState`, like the
existing paging/timeline logic; `indexCfg.formatter` sees the absolute number.

## Added: `disabled` for row actions

`actionCfg.actions[].disabled: row => boolean` keeps a row action visible but
greyed out — for both direct actions and menu triggers (a disabled trigger does
not open its menu). When `icon.tooltip` is set, the tooltip renders on a wrapper
element so it still shows on the disabled button (disabled buttons swallow
pointer events) — useful for explaining why the action is unavailable.

```ts
actionCfg: { actions: [{ action: r => 'edit', icon: { name: () => 'edit', tooltip: () => 'system row' }, disabled: r => r.system }] }
```

## Fixed: `IconView.visible: false` renders nothing

A view icon with `visible: false` used to leave an empty wrapper `<div>` — and,
with `wrapper` set, a filled coloured circle with no icon — because the
visibility check only gated the `<mat-icon>`. Visibility now gates the outer
element, so nothing renders. Affects cell/header icons and the drag icon.

## Added: per-row hover, pointer and click styling

`hoverCfg` and `clickCfg` are now generic (`HoverConfig<T>` / `ClickConfig<T>`),
and `pointer`, `styleCfg.style` and `styleCfg.class` accept a static value **or**
a `(row: TableRow<T>) => value` function (new exported type `RowValue<T, R>`).
So hover, the pointer cursor and the click-highlight can be disabled or varied
per row — e.g. turned off for system rows — while the rest of the table stays
interactive:

```ts
bodyRowCfg: {
  hoverCfg: {
    pointer: row => !row.rowSrc.system,
    styleCfg: { style: row => row.rowSrc.system ? '' : 'background:#f5f5f5' },
  },
}
```

The function receives the `TableRow<T>` (access the source via `row.rowSrc`),
matching `bodyRowCfg.styleCfg`. `styleCfg.class` may now return `null` to opt a
row out of the class. Static config is unchanged; `enable`/`cancelable` stay
table-wide.

## Added: `tooltipPosition` for icons, actions and cell text

`IconView.tooltipPosition` and `TextView.tooltipPosition` set Material's
`matTooltipPosition` (`'left' | 'right' | 'above' | 'below' | 'before' |
'after'`, default `'below'`) wherever a tooltip is rendered from a view: column
icons, the drag icon, cell text, row-action buttons and selection actions.
Useful for narrow or edge columns. `tooltipClass` was already supported; this
adds the position alongside it.

```ts
valueView: { icon: { name: () => 'info', tooltip: () => 'hint', tooltipPosition: 'right' } }
```

## Added: table-wide default alignment — `tableViewCfg.align`

`tableViewCfg.align` (`'left' | 'center' | 'right'`) sets the default alignment
for all data columns **and** the index column at once, so a uniformly-aligned
table no longer repeats `align` on every column. A per-column
`ColumnConfig.align` / `IndexConfig.align` overrides it (an explicit `'left'`
resets a column back to left even under a `center` table default). Special
columns (selection/action/drag/timeline) are unaffected. Without it columns stay
left-aligned as before.

```ts
tableViewCfg: { align: 'center' },
columnsCfg: [{ key: 'name', name: 'Name', valueConverter: v => v.name, align: 'left' }],
```
