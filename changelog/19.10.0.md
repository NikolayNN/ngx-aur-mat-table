# 19.10.0

Breaking release: all four template inputs (`@Input` `TemplateRef` bindings) are
replaced by content-projected structural directives. Templates are now delivered
inside `<aur-mat-table>…</aur-mat-table>` rather than as sibling references, and
the row-template context (`$implicit`) now exposes the source object directly
instead of the `TableRow` wrapper. Two new types are exported. Suite green
(225 tests), library and demo build clean.

## Changed / Removed (breaking): template inputs → projection directives

The four `@Input` bindings that accepted a `TemplateRef` are removed. Each is
replaced by a matching content-projection directive placed **inside** the table
element:

| Removed `@Input` | New directive |
|---|---|
| `[extendedRowTemplate]` | `ngxAurExpandedRowDef` |
| `[timelineMarkerTemplate]` | `ngxAurRowMarkerDef` |
| `[extraHeaderCellTopTemplate]` | `ngxAurExtraHeaderTopDef` |
| `[extraHeaderCellBottomTemplate]` | `ngxAurExtraHeaderBottomDef` |

Before (≤ 19.9.x):

```html
<aur-mat-table [tableData]="data" [tableConfig]="cfg"
               [extendedRowTemplate]="detailTpl">
</aur-mat-table>
<ng-template #detailTpl let-row>
  <div>{{ row.rowSrc.name }}</div>
</ng-template>
```

After (19.10.0):

```html
<aur-mat-table [tableData]="data" [tableConfig]="cfg">
  <ng-template ngxAurExpandedRowDef let-rowSrc>
    <div>{{ rowSrc.name }}</div>
  </ng-template>
</aur-mat-table>
```

The same pattern applies to `ngxAurRowMarkerDef`, `ngxAurExtraHeaderTopDef`, and
`ngxAurExtraHeaderBottomDef`.

## Changed (breaking): row-template context — `$implicit` is now the source object

For the two row-level directives (`ngxAurExpandedRowDef`, `ngxAurRowMarkerDef`),
the template context changes:

- **Before:** `$implicit` was the `TableRow<T>` wrapper (access source via `row.rowSrc`).
- **After:** `$implicit` is the source object `T` directly. The full wrapper is still
  reachable as the named binding `row`.

```html
<!-- before: $implicit was the wrapper -->
<ng-template #tpl let-row>{{ row.rowSrc.name }}</ng-template>

<!-- after option A: use $implicit as the source directly -->
<ng-template ngxAurExpandedRowDef let-rowSrc>{{ rowSrc.name }}</ng-template>

<!-- after option B: bind the wrapper explicitly and keep existing access pattern -->
<ng-template ngxAurExpandedRowDef let-row="row">{{ row.rowSrc.name }}</ng-template>
```

The extra-header directives (`ngxAurExtraHeaderTopDef`, `ngxAurExtraHeaderBottomDef`)
are unaffected — their context remains `{ key: string; index: number }`.

## Added: exported types `AurRowContext<T>` and `AurExtraHeaderContext`

Both context shapes are now exported from the public API:

```ts
// row directives (ngxAurExpandedRowDef, ngxAurRowMarkerDef)
interface AurRowContext<T> {
  $implicit: T;          // source object (rowSrc)
  row: TableRow<T>;      // full wrapper (row.rowSrc, row.id, …)
  rowSrc: T;             // alias for $implicit
  index: number;
}

// extra-header directives (ngxAurExtraHeaderTopDef, ngxAurExtraHeaderBottomDef)
interface AurExtraHeaderContext {
  key: string;
  index: number;
}
```
