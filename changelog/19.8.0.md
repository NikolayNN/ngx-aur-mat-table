# 19.8.0

Single additive feature, no breaking changes: a column can now render its data
cells with a custom Angular template when the declarative `valueView`
(icon/image/text) is not expressive enough. Suite green (195 tests), library and
demo build clean.

## Added: custom cell template — `ngxAurCellDef`

A projection directive lets the host render a column's data cells with an
arbitrary `<ng-template>`, placed inside `<aur-mat-table>` and bound to a column
by its `key`:

```html
<aur-mat-table [tableData]="data" [tableConfig]="cfg">
  <ng-template ngxAurCellDef="status" let-value let-row="row" let-i="index">
    <span class="chip" [class.on]="row.rowSrc.active">{{ value }} (#{{ i }})</span>
  </ng-template>
</aur-mat-table>
```

The template context exposes `$implicit`/`value` (the column's `valueConverter`
result), `row` (the `TableRow` — `row.rowSrc` is the source object, `row.id` the
row index), `rowSrc` (alias of `row.rowSrc`) and `index`. A custom template takes
priority over `valueView`, and a column without a matching template falls back to
the existing `lib-column-view` / plain `<span>`. Templates may be added or removed
at runtime (e.g. behind `*ngIf`); an `ngxAurCellDef` key that matches no column
logs a dev-mode warning.

It replaces only the data-cell body — the header (`headerView`) and total row
(`totalConverter`) are unchanged — and `valueConverter` stays required, so search,
sorting and the total row keep working off the column value exactly as before. The
`<td>` wrapper (size, padding, alignment) is untouched.
