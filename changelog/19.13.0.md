## Added: row type marker classes — `aur-data-row` / `aur-expanded-row` / `aur-total-row`

Each rendered `<tr>` gets a namespaced marker class identifying its type, giving
consumers a reliable hook to distinguish and style the three row types from their
own (global / `::ng-deep`) CSS:

| Row type | Marker class |
|---|---|
| data row | `aur-data-row` |
| detail / expanded row (`ngxAurExpandedRowDef`) | `aur-expanded-row` |
| total / footer row | `aur-total-row` |

The library attaches no styling of its own to these classes — they are pure hooks
(neutral posture). The detail row keeps its legacy `expanded-row` class, so existing
selectors and the expand animation are unaffected. The markers are disjoint: each
appears only on its own `<tr>`.

A common use is neutralizing the hover an app/theme applies to the detail row, or
giving the detail "drawer" its own background — things the projected
`ngxAurExpandedRowDef` content can't reach, because they live on the row `<tr>`,
not on the cell inside it:

```css
/* neutralize inherited hover on the detail row */
tr.aur-expanded-row:hover { background: none; }

/* give the detail "drawer" a background and drop its own bottom border */
tr.aur-expanded-row { background: #fafafa; }
tr.aur-expanded-row td.expanded-cell { border-bottom: none; }
```

No new public TS exports; the feature is entirely CSS-level. Merging a detail row
visually with its parent (removing the separator *between* them) is the parent data
row's cell `border-bottom` and stays out of scope.
