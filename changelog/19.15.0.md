# 19.15.0

Additive feature: controlled row highlight via `highlightCfg` and a two-way
`[(highlightedRow)]` binding. **Breaking removal** in the same release: the legacy
`[highlight]`/`HighlightContainer` input and `clickCfg.cancelable`/`clickCfg.styleCfg`
fields have been removed; use `highlightCfg` / `[(highlightedRow)]` instead.
Suite green, library and demo build clean.

## Added: controlled row highlight — `highlightCfg` and two-way `[(highlightedRow)]`

Row highlight is now configured through `bodyRowCfg.highlightCfg`, with three ownership
modes that decouple highlight from the click itself:

| mode | state owner | click highlights | `(highlightedRowChange)` | `(rowClick)` |
|---|---|---|---|---|
| `row-click` (default) | table | yes | yes | yes |
| `controlled` | `[highlightedRow]` | no (emits a request) | yes | yes |
| `manual` | `[highlightedRow]` | no | no | yes |

`highlightCfg` accepts:
- `mode` — one of the three modes above (defaults to `'row-click'`, the previous behaviour).
- `cancelable: true` — re-clicking the highlighted row clears the highlight.
- `styleCfg` — `{ style, class }` with the same rules as `bodyRowCfg.styleCfg`.

`manual` mode answers "highlight is not a row click": the click still fires `(rowClick)`
for actions, but never changes the highlight. `controlled` mode lets a parent own the
state — the click emits `(highlightedRowChange)` without mutating internal state, and the
parent applies it back through `[highlightedRow]` (the two-way `[(highlightedRow)]` closes
the loop).

```html
<aur-mat-table [tableConfig]="cfg" [tableData]="data"
               [(highlightedRow)]="selected"></aur-mat-table>
```

```ts
bodyRowCfg: {
  highlightCfg: {
    mode: 'controlled',
    cancelable: true,
    styleCfg: { class: 'is-selected', style: StyleBuilder.Row.builder().background('blue') },
  },
}
```

## Removed (breaking): `[highlight]` / `clickCfg.cancelable` / `clickCfg.styleCfg`

- `[highlight]` / `HighlightContainer<T>` — removed. Use `[(highlightedRow)]` + `highlightCfg`.
- `clickCfg.cancelable` — removed. Use `highlightCfg.cancelable`.
- `clickCfg.styleCfg` / `ClickStyleConfig<T>` — removed. Use `highlightCfg.styleCfg`.

Migration: replace `bodyRowCfg.clickCfg: { styleCfg, cancelable }` with
`bodyRowCfg.highlightCfg: { styleCfg, cancelable }` (the sub-keys are identical).
