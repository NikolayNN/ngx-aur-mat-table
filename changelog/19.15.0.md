# 19.15.0

Additive feature: controlled row highlight via `highlightCfg` and a two-way
`[(highlightedRow)]` binding. No breaking changes — the old `[highlight]`/`HighlightContainer`
path and `clickCfg.cancelable`/`clickCfg.styleCfg` continue to work as a fallback with a
deprecation warning. Suite green (287 tests), library and demo build clean.

## Добавлено: `highlightCfg` и two-way `[(highlightedRow)]`

Подсветка строки теперь управляется через `bodyRowCfg.highlightCfg`. Поддерживаются три режима:

| mode | владелец состояния | клик подсвечивает | `(highlightedRowChange)` | `(rowClick)` |
|---|---|---|---|---|
| `row-click` (по умолчанию) | таблица | да | да | да |
| `controlled` | `[highlightedRow]` | нет (эмит-запрос) | да | да |
| `manual` | `[highlightedRow]` | нет | нет | да |

`highlightCfg` принимает:
- `mode` — один из трёх режимов выше (по умолчанию `'row-click'`).
- `cancelable: true` — повторный клик по подсвеченной строке снимает подсветку.
- `styleCfg` — `{ style, class }` с теми же правилами, что у `bodyRowCfg.styleCfg`.

Two-way binding `[(highlightedRow)]` позволяет синхронизировать подсвеченную строку с
внешним состоянием компонента-потребителя. Одностороннее `[highlightedRow]` и
`(highlightedRowChange)` доступны по отдельности.

```html
<aur-mat-table [tableConfig]="cfg" [tableData]="data"
               [(highlightedRow)]="selected"></aur-mat-table>
```

```ts
bodyRowCfg: {
  highlightCfg: {
    mode: 'controlled',
    cancelable: true,
    styleCfg: { class: 'is-selected', style: StyleBuilder.Row.builder().background('blue') },
  },
}
```

## Устарело

- `[highlight]` / `HighlightContainer` — заменяются на `[(highlightedRow)]` + `highlightCfg`.
- `clickCfg.cancelable` — переезжает в `highlightCfg.cancelable`.
- `clickCfg.styleCfg` — переезжает в `highlightCfg.styleCfg`.

Все три продолжают работать как fallback с выводом `console.warn`. Удаление запланировано
на следующий мажорный релиз.
