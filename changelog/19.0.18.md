## ngx-aur-mat-table v19.0.18

**Новое: декларативная серверная пагинация через `pageSource`**

Добавлен рекомендуемый способ серверной пагинации — входной параметр `pageSource`. Таблица самостоятельно выполняет начальную загрузку и повторные запросы при смене страницы или сортировки.

### Новые возможности

- **`pageSource`** — новый входной параметр типа `AurPageSource<T>`. Принимает функцию-загрузчик `(req) => Observable<AurPage<T>>`. `AurPage<T>` совместим с форматом Spring Data `Page<T>` (`content`, `totalElements`, опционально `number`), что позволяет передавать ответ бэкенда без маппинга.

- **`PaginationConfig.mode`** — новое поле `'client' | 'server'`. При `mode: 'server'` таблица переключается в режим серверной пагинации совместно с `pageSource`.

- **`PaginatorState.of({ total, pageIndex })`** — новый статический фабричный метод для удобного создания состояния пагинатора.

- **`externalPaginator`** — новый входной параметр для привязки внешнего `<mat-paginator>`, управляемого хост-компонентом. Работает с клиентскими данными (`tableData`) и серверными (`pageSource`).

- **`reload(opts?)`** — новый публичный метод компонента `NgxAurMatTableComponent`. Сбрасывает страницу на 0 и повторно выполняет загрузку через `pageSource`. Используется при изменении внешних фильтров.

- **`loadingChange`** — новый выходной параметр (`EventEmitter<boolean>`), сигнализирующий о начале и завершении загрузки.

- **`pageError`** — новый выходной параметр (`EventEmitter<unknown>`), передающий ошибку, возникшую при загрузке страницы.

### Устаревшее (deprecated)

- `NgxAurTablePageEventUtils.createEmpty` — помечен как устаревший в пользу `pageSource`. Ручная связка `[paginatorState]` + `(pageChange)` + `NgxAurTablePageEventUtils.createEmpty` продолжает работать, но рекомендуется перейти на `pageSource`.

### Обратная совместимость

Изменения полностью аддитивны. Существующие варианты клиентской и серверной пагинации работают без изменений.

```typescript
// Рекомендуемый способ (pageSource)
const tableConfig: TableConfig<Customer> = {
  columnsCfg: [/* ... */],
  pageableCfg: { enable: true, size: 20, mode: 'server' }
};

loadPage: AurPageSource<Customer> = req =>
  this.customerService.page(req.pageIndex, req.pageSize);
```

```html
<aur-mat-table #table
  [tableConfig]="tableConfig"
  [pageSource]="loadPage"
  (loadingChange)="loading = $event"
  (pageError)="onError($event)">
</aur-mat-table>
```

При изменении внешнего фильтра вызывайте `table.reload()`:

```typescript
@ViewChild('table') table!: NgxAurMatTableComponent<Customer>;
onFilterChange() { this.rebuildFilters(); this.table.reload(); }
```
