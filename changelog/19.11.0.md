## Added: custom header template â€” `ngxAurHeaderCellDef`

A projection directive lets the host render a column's `<th>` header with an
arbitrary `<ng-template>`, placed inside `<aur-mat-table>` and bound to a column
by its `key`:

```html
<aur-mat-table [tableData]="data" [tableConfig]="cfg">

  <!-- default: template renders inside mat-sort-header (built-in arrow + click kept) -->
  <ng-template ngxAurHeaderCellDef="amount" let-column let-sort="sort">
    <b>{{ column.name }}</b> <small *ngIf="sort.active">{{ sort.direction }}</small>
  </ng-template>

  <!-- ownsCell: template owns the whole <th> (no mat-sort-header); sort/filter from context -->
  <ng-template ngxAurHeaderCellDef="status" ownsCell let-filter="filter">
    <input (input)="filter.apply(buildFilter($event))" placeholder="filter">
    <button *ngIf="filter.active" (click)="filter.remove()">âś•</button>
  </ng-template>

</aur-mat-table>
```

The template context exposes `$implicit`/`column` (the column's `ColumnConfig`),
`key`, a `sort` handle (`{ sortable, active, direction, toggle() }`) and a `filter`
handle (`{ apply(filter), remove(), active }` â€” a thin wrapper over the public
`applyFilter`/`removeFilter`, keyed by the column). A custom template takes
priority over `name`/`headerView`, and a column without a matching template falls
back to the existing `lib-column-view`. Templates may be added or removed at
runtime (e.g. behind `*ngIf`); an `ngxAurHeaderCellDef` key that matches no column
logs a dev-mode warning.

Sort coexistence is configurable per column. By default the template renders
**inside** `mat-sort-header`, so the built-in sort arrow and whole-cell click are
preserved (suited to cosmetic headers). For interactive content (filter input,
selector, toggle) add the `ownsCell` attribute: it drops `mat-sort-header` and
hands the whole `<th>` to the template, with sorting rebuildable through the `sort`
handle. Without a projected template, header rendering is byte-identical to before.

It targets data columns only â€” special-column headers (selection / index /
actions) and the total row stay out of scope (use their built-in settings and
`totalConverter`). The `<th>` wrapper (size, padding, alignment) is untouched.
