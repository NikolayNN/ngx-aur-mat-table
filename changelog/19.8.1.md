# 19.8.1

Single additive feature, no breaking changes: rows can now be made fully
non-interactive via an explicit opt-out flag. Default behavior is unchanged.
Suite green (201 tests), library builds clean.

## Added: row click opt-out — `clickCfg.enable`

`bodyRowCfg.clickCfg.enable: false` turns a table's rows fully non-interactive in
one switch:

```ts
bodyRowCfg: {
  clickCfg: {
    enable: false,
  },
},
```

With `enable: false` the row click handler does not run, `rowClick` is not
emitted, the internal highlight does not change, the detail row is not
auto-expanded on click, and the row gets no interactive `tabindex` / keyboard
activation. Until now the presence of a table implied an interactive row — the
mouse-click path fired even without any `clickCfg`, and there was no way to
decouple highlight, expansion and `rowClick`.

The flag follows the library's opt-out `enable` convention: rows stay interactive
by default; only `enable: false` disables them. Programmatic highlighting via the
`[highlight]` input is unaffected (it bypasses the click handler), so an
externally-driven detail row still expands. No `role` is added to the row — a
`<tr mat-row>` already carries Material's implicit grid `row` role.
