# 19.9.0

Breaking + additive release: detail-row expansion is decoupled from highlight
into its own independently controlled state. Three expansion modes and
multi-row support are added; `[highlight]` no longer expands the detail row.
Suite green (209 tests), library and demo build clean.

## Added: `extendedRowCfg` â€” expansion modes and multi-row support

A new `extendedRowCfg` object on `TableConfig` governs how the detail row
(`extendedRowTemplate`) is opened and closed:

```ts
extendedRowCfg?: {
  mode?: 'row-click' | 'controlled' | 'manual';  // default: 'row-click'
  multiple?: boolean;                              // default: false
}
```

`mode` determines who owns the expansion state:

- **`row-click`** (default) â€” the table opens/collapses on click, no host
  wiring needed. Re-clicking an already-expanded row collapses it.
- **`controlled`** â€” the host is the source of truth; the table emits
  `(expandedRowChange)` / `(expandedRowsChange)` on click and applies the
  value received through `[expandedRow]` / `[expandedRows]`.
- **`manual`** â€” state flows in from the host inputs only; clicks do not
  trigger expansion.

`multiple: true` allows several rows to be open at the same time.

## Added: `[expandedRow]` / `[expandedRows]` inputs and `(expandedRowChange)` / `(expandedRowsChange)` outputs

Single-row mode (`multiple: false`) uses `[expandedRow]: T | null` and
`(expandedRowChange): T | null`.  
Multi-row mode (`multiple: true`) uses `[expandedRows]: T[]` and
`(expandedRowsChange): T[]`.

Two-way binding works with the `[()]` banana-in-a-box syntax in both cases:

```html
<!-- single -->
<aur-mat-table [extendedRowTemplate]="tpl" [(expandedRow)]="openRow" ...></aur-mat-table>

<!-- multiple -->
<aur-mat-table [extendedRowTemplate]="tpl" [(expandedRows)]="openRows"
               [extendedRowCfg]="{ mode: 'controlled', multiple: true }" ...></aur-mat-table>
```

## Added: expansion identity via `trackBy` (survives server reload)

The expanded-row state is mapped through `tableConfig.trackBy` when set.
After a server reload the previously expanded row re-opens automatically if
its identity (tracked key) is still present in the new data. Without
`trackBy` identity falls back to object reference.

## Changed: re-click collapses in `row-click` mode regardless of `clickCfg.cancelable`

Previously a second click on an already-expanded row would leave it open
when `clickCfg.cancelable` was `false` (or absent). Now the re-click always
collapses the detail row in `row-click` mode. `cancelable` continues to
govern highlight deselection and the `rowClick` event â€” it no longer
influences expansion.

## Removed (breaking): `[highlight]` no longer expands the detail row

`[highlight]` now only highlights the row and scrolls it into view.
Programmatic expansion must be migrated to `[expandedRow]` / `[expandedRows]`.

See `docs/MIGRATION-19.9.0.md` for a step-by-step migration guide.
