## ngx-aur-mat-table v19.0.17

**Новое: выпадающее меню на иконке в колонке действий**

У каждого действия в `actionCfg` появилось опциональное поле `menu`. Если оно задано, иконка действия становится триггером `mat-menu` и по нажатию открывает список пунктов вместо немедленной генерации события. Клик по пункту меню эмитит то же событие `onRowAction` со своим кодом действия.

```typescript
const tableConfig: TableConfig<Customer> = {
  columnsCfg: [/* ... */],
  actionCfg: {
    actions: [
      {
        action: () => 'menu',
        icon: { name: () => 'more_vert', tooltip: () => 'действия' },
        menu: [
          {
            action: () => 'view',
            text: () => 'Просмотр',
            icon: { name: () => 'visibility', color: () => 'blue' }
          },
          {
            action: () => 'archive',
            text: () => 'Архивировать',
            icon: { name: () => 'archive' },
            // условное отключение пункта по данным строки
            disabled: (c) => (c.age < 18 ? 'true' : 'false')
          },
          {
            action: () => 'block',
            text: () => 'Заблокировать',
            // условное скрытие пункта по данным строки
            display: (c) => (c.age < 18 ? 'none' : 'show')
          },
          {
            action: () => 'delete',
            text: () => 'Удалить'
          }
        ]
      }
    ]
  }
};
```

Поля пункта `MenuItem` (как и у остальной конфигурации — функции `(value) => string`, вычисляемые для каждой строки):

- `action` — код действия, уходит в `onRowAction`;
- `text` — подпись пункта;
- `icon` — опциональная иконка (`IconView`);
- `display` — `'show'` | `'none'`, условное скрытие пункта;
- `disabled` — `'true'` | `'false'`, условное отключение пункта.

Поле `menu` опционально и полностью обратно совместимо: без него действие работает как раньше (прямой `onRowAction`).
