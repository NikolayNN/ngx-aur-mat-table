# 19.5.0

Performance/footprint release: the remaining quick wins from the core
performance review. No API changes; a few behavior notes below.

## Performance: lighter cells, fewer idle directives

- Columns without `valueView` now render a bare `<span class="aur-plain-cell">`
  instead of the `lib-column-view` + `lib-icon-view` component pair — 2 component
  instances, 4 wrapper nodes and a flex context less **per cell**. The span keeps
  the same 4px text offset, so plain cells stay pixel-aligned with headers.
- View columns without an icon no longer instantiate an empty `IconView`;
  `lib-icon-view` renders nothing at all when it has no view.
- `MatTooltip` is instantiated only where a tooltip is actually configured
  (cell text, icons, per-row action buttons). Previously every cell carried an
  idle tooltip directive (DI, `dir.change` subscription, FocusMonitor
  registration each) — 500+ instances on a 50×10 table with no tooltips at all.
- `ColumnViewComponent` and `IconViewComponent` switched to `OnPush`: a table
  change-detection cycle (hover, click, search keystroke) no longer re-evaluates
  every cell template (~1000 on a 50×10 table); cells re-render when their view
  objects are rebuilt by a data refresh, as before.
- Column defs of optional features (extra header top/bottom cells — 2 per column,
  `expandedRow`, `subFooterRow`) are registered with the table only when their
  templates are actually provided.
- `TableViewFactory` no longer allocates an empty per-row `Map` on every refresh
  when no column defines `valueView`.
- `updateColumnOffsets` runs a single `querySelectorAll` instead of two.

## Bundle: unused CDK DragDropModule removed

The library's drag-and-drop is built on native HTML5 events and its own
`AurDragDropManager`; no CDK drag-drop directive was ever referenced. The
leftover `DragDropModule` import forced `@angular/cdk/drag-drop` into every
consumer bundle: ~66 KB minified (~15.8 KB gzip) saved for apps that do not
use CDK drag-and-drop themselves.

## SSR: importing the library no longer crashes in Node

`EmptyValue` created a `ResizeObserver` at class-initialization time, which
threw on import under Node/SSR. The observer is now an optional field created
lazily in `ngAfterViewInit` behind a `typeof ResizeObserver` guard — the
deprecated `columnOffsets` feature simply stays inactive on the server.

## Behavior notes

- With `OnPush` cell components, mutating a consumer-owned `headerView` object
  **in place** is no longer reflected between refreshes. Provide a new config
  object and refresh instead (the supported path, unchanged). Cell view objects
  are library-built from resolvers and cannot be mutated externally.
- Styles that targeted `lib-column-view` / `.align-container` of **plain**
  columns via `::ng-deep` no longer match: plain cells render a bare
  `span.aur-plain-cell`. Columns with `valueView` keep the previous structure.
