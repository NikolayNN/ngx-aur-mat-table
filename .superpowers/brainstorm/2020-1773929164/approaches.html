Подходы к стилизации таблицы

Все варианты: скругление 8-12px, граница #bdbdbd, padding 12px, жирная шапка с фоном

A

CSS-only в SCSS компонента

Все стили добавляются напрямую в ngx-aur-mat-table.component.scss. Используем border-collapse: separate + border-spacing: 0 и скругляем через border-radius на обёртке .table-container.

Плюсы

  • Простейшая реализация — только CSS изменения
  • Не ломает существующую конфигурацию
  • Все пользователи библиотеки получают обновлённый стиль

Минусы

  • Нет возможности отключить новые стили — breaking change для тех, кто привык к старому виду
  • Сложно кастомизировать (радиус, цвет границы)
B

CSS-only + CSS-переменные

Те же CSS изменения, но ключевые значения вынесены в CSS custom properties с дефолтами. Пользователи могут переопределять через :host или родительский элемент.

Плюсы

  • Простая реализация — только CSS
  • Кастомизируемо: --aur-table-border-radius, --aur-table-border-color, --aur-table-cell-padding
  • Можно "отключить" скругление, задав переменную в 0

Минусы

  • Всё равно меняет дефолтный вид (но это и есть цель)
  • Чуть больше CSS кода
C

Конфигурация через TableConfig

Добавляем новое поле styleCfg в TableConfig с параметрами borderRadius, borderColor, cellPadding, headerBackground. Стили применяются через [ngStyle] в шаблоне.

Плюсы

  • Полностью контролируемо из TypeScript
  • Вписывается в существующий подход конфигурации

Минусы

  • Избыточно для чисто визуальных свойств — CSS-переменные делают то же проще
  • Больше кода: интерфейс, дефолты, шаблонные привязки
  • Смешивает стилевые и логические настройки

Рекомендация: Вариант B — CSS-переменные дают баланс между простотой и гибкостью. Это чисто стилевые изменения, и CSS custom properties — идиоматический способ кастомизации в Angular компонентах.
